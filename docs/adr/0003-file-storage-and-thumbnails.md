# ADR-0003: 파일 저장과 썸네일 생성 (MinIO + presigned + 서버 썸네일)

- 상태: 승인 대기
- 날짜: 2026-06-23

## 컨텍스트

일반 게시판은 첨부 업로드/다운로드, 이미지 게시판은 다중 이미지 + 썸네일 카드 그리드 + 원본 보기가 필요하다.

## 결정

- 원본/썸네일 바이너리는 MinIO(S3 호환)에 저장, 메타데이터(`storage_key`/`thumbnail_key` 등)는 DB.
- 업로드/다운로드는 **presigned URL**(단기 만료)로 처리.
- 썸네일은 **업로드 시 서버(Pillow)에서 생성**해 별도 키로 저장.

## 대안

- 파일을 DB(BLOB)에 저장: 확장성/성능 불리.
- 썸네일을 프론트 CSS로 축소: 대용량 이미지에서 대역폭 낭비, 목록 성능 저하.
- 온디맨드 썸네일 생성: 첫 조회 지연. 업로드 시 생성으로 조회 일관 성능 확보.

## 결과

- 목록(그리드)은 가벼운 썸네일, 상세는 원본 — 성능·UX 우수.
- DB-S3 정합성 관리 필요(업로드 트랜잭션 경계, orphan 정리 — data.md 참조).
- 이미지 재인코딩으로 악성 파일 위험 완화(security.md).
