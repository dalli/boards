# 에이전트 지침서 — 경량판 (AGENTS-light.md)

이 문서는 **소규모 프로젝트**(1~2인, 프로토타입·학습·내부 도구, 규제 요건 없음)를 위한 경량 거버넌스입니다. 풀 버전 [AGENTS.md](AGENTS.md)에서 **가치 대비 비용이 가장 높았던 규칙만 남기고**, 산출물 강제는 면제하거나 선택으로 돌렸습니다.

> **선정 기준(보드 프로젝트 회고)**: 실제로 결함을 잡은 것은 **독립 교차 리뷰 + 콘솔 증거 강제**였습니다. 반면 ADR·시퀀스 다이어그램·다중 모델 감사 추적은 소형 프로젝트에선 과투자였고, 자동 강제가 없던 체크박스 게이트는 끝에서 빠졌습니다. 경량판은 이 교훈을 반영합니다.

> **승급 규칙**: 아래 중 하나라도 해당되면 경량판을 쓰지 말고 풀 [AGENTS.md](AGENTS.md)로 전환합니다 — 3인 이상 협업, 외부 사용자/PII·결제 데이터, 규제·컴플라이언스 요건, 되돌리기 어려운 아키텍처 결정이 다수.

## 0. 프로젝트 선언 (Declaration)

에이전트는 가정을 하드코딩하지 말고 아래 선언을 참조합니다. 선언이 없어 검증할 수 없으면 추측하지 말고 **§4 이스케이프**.

- **버전 관리**: git 저장소 전제(`git rev-parse --is-inside-work-tree` → `true`). 초기화 안 돼 있으면 임의 `git init` 금지 → §4.
- **명령 선언**: 실행/테스트/린트 명령을 아래 표에 바인딩합니다. 에이전트는 명령을 추측하지 않습니다.
- **작업 스코프**: 수정 가능한 디렉토리/모듈 경계를 선언합니다.
- **통합 브랜치**: 작업 결과를 통합할 브랜치를 선언합니다(기본 `main` 또는 `dev`).

| 목적 | 플레이스홀더 | 이 프로젝트 바인딩 |
| --- | --- | --- |
| 실행 | `<RUN_CMD>` | _(예: `npm run dev`)_ |
| 테스트 | `<TEST_CMD>` | _(예: `npm test`)_ |
| 린트/타입체크 | `<LINT_CMD>` | _(예: `npm run lint`)_ |
| 커버리지 게이트(선택) | `<COVERAGE_CMD>` | _(예: `npm run test:coverage`)_ |
| 통합 브랜치 | `<INTEGRATION_BRANCH>` | _(예: `main`)_ |
| 작업 스코프 | `<SCOPE>` | _(예: `src/`)_ |

> 명령이 실제로 실행 가능한지 먼저 확인합니다. 정의돼 있지 않으면 임의 대체 금지 → §4.

## 1. 역할 (Roles)

작은 팀이므로 역할은 최소화합니다. 한 사람/에이전트가 여러 역할을 겸할 수 있되, **리뷰만은 구현 주체와 독립**이어야 합니다(§3.2 핵심).

- **Planner/Implementor**: 계획을 세우고 구현합니다. (겸임 허용)
- **Independent Reviewer**: 구현을 **다른 모델 또는 독립 세션**으로 리뷰합니다. **자기 자신(동일 agent/session/model)의 리뷰는 무효.**

## 2. 계획 (Plan) — 경량

코드 작성 전 **한 개의 계획 문서**(`plan.md`)만 작성하면 됩니다. 풀 버전의 아키텍처 산출물 7종은 **면제**하되, 아래만 포함합니다.

1. **목표**와 **수락 기준(AC)**: 각 AC는 검증 가능한 관찰 결과(상태 코드/출력/테스트)로 기술하고 `- [ ]` 체크박스로 적습니다.
2. **phase 분해**: 큰 작업을 검증 가능한 작은 단위로 나눕니다(번호·목표·AC).
3. **설계 메모(선택)**: 비자명한 구조나 트레이드오프가 있을 때만 짧게 적습니다. **다이어그램 강제 없음.**
4. **인간 승인**: 계획을 인간에게 보여주고 **명시적 승인 후** 구현 시작. (1인 프로젝트면 본인 확인으로 갈음)

> **ADR·시퀀스/데이터/배포 다이어그램은 기본 면제.** 단, **되돌리기 어려운 결정**(DB 선택, 인증 방식, 외부 의존 도입)은 `plan.md`에 3~5줄짜리 결정 메모(컨텍스트·결정·대안)를 남깁니다.

## 3. 완료의 정의 (Definition of Done)

모든 완료는 **"통과했다"는 서술이 아니라 콘솔 출력 증거**로 입증합니다.

### 3.1. 구현 완료
- 승인된 AC를 충족하고, **AC 체크박스를 실제로 채웁니다.** (아래 §6 hook으로 자동 강제 권장)
- 커밋 대상 diff에 `TODO`/`FIXME`/`HACK` 등 임시 코드·데드 코드가 없습니다.
- 할당된 `<SCOPE>` 밖 파일을 수정하지 않습니다.

### 3.2. 테스트 & 리뷰 완료 (★ 핵심 게이트)
1. **테스트 통과**: `<TEST_CMD>` 실패 0건. 콘솔 출력 첨부.
2. **정적 분석 통과**: `<LINT_CMD>` 경고·타입 에러 0건. 콘솔 출력 첨부.
3. **신규 로직엔 테스트 존재**: 이번 변경의 신규/변경 로직에 대응 테스트가 있어야 합니다. 커버리지 게이트를 쓴다면 `<COVERAGE_CMD>`로 종료 코드 판정.
4. **독립 교차 리뷰**: 구현을 **독립 리뷰어**(다른 모델/세션)에게 1회 이상 리뷰받습니다.
   - 리뷰 결과를 `docs/reviews/phase-<N>-review.md`에 기록: 각 지적은 **ID·severity·파일/라인·요약**.
   - **blocking/high 지적은 수정 또는 인간 판정(Rejected 근거 명시) 전까지 통합 불가.**
   - 수정 후 **재리뷰**하여 미해결 blocking이 없음을 확인합니다.

> 보드 프로젝트에서 이 게이트가 동시성 갱신 유실·인가 우회 같은 **머지됐을 실제 버그**를 잡았습니다. 경량판에서도 이건 **타협하지 않습니다.**

## 4. 이스케이프 밸브 (Escalation)

다음 상황이면 추측하지 말고 **즉시 중단하고 인간에게 질문**합니다.

1. 동일 테스트 실패/빌드 에러가 **3회 이상** 반복되며 해결 안 됨.
2. **시크릿/자격증명**(`.env` 등), DB 마이그레이션, CI/CD 설정 수정 필요.
3. **모호성**: 권한·암호화 등 미정의 요구사항, 승인된 계획과 충돌.
4. **환경 미비**: git 미초기화, 선언된 명령 부재.
5. **보호 파일 변경**: `AGENTS.md`/`AGENTS-light.md`, 승인된 AC, 위 2항 파일.

> 에스컬레이션은 **인간만** 해소할 수 있고, **검토 대상의 작성자(에이전트 포함)는 승인자를 대체할 수 없습니다.** 인간 무응답이 길어지면 상태를 `BLOCKED`로 기록하고 멈춥니다(임의 진행 금지).

## 5. 스코프 & 커밋 (Scope & Commits)

- **스코프 준수**: 할당된 `<SCOPE>` 밖을 수정하지 않습니다. 공유 계약(API 스펙·공유 타입)을 바꿔야 하면 계획에 명시·승인 후 진행.
- **원자적 커밋**: 하나의 커밋은 하나의 논리 변경. 무관한 변경 섞지 않기.
- **커밋 메시지**: 명령형 현재 시제로 *이유* 설명.
- **검증 후 커밋**: §3의 테스트·린트·리뷰 게이트 통과·증거 확보 후에만 커밋.
- **자동 커밋·푸시 금지**: phase 종료 시 `<INTEGRATION_BRANCH>` 통합을 제외하고, 인간 요청 전까지 임의 커밋/푸시 금지.

## 6. phase 종료 게이트 (자동 강제 권장)

각 phase는 **구현 → 테스트/린트 → 독립 리뷰 → 수정·재리뷰 → 통합** 순으로 닫습니다. 회고는 **선택**이며, 남긴다면 한 단락(한 일/막힌 점/인간 확인 필요 항목)이면 충분합니다.

> **회고에서 얻은 교훈**: AC 체크박스 게이트는 **사람 의지에 맡기면 빠집니다.** 아래를 pre-commit hook 또는 CI로 강제하세요.

```bash
# AC 미체크 차단 — 매칭이 없어야 통과
if grep -nE '^\s*- \[ \]' plan.md; then
  echo "❌ 미충족 AC 존재 → 완료 아님"; exit 1
else
  echo "✅ 모든 AC 체크 완료"
fi
```

```bash
# phase 종료 게이트 (실패 시 non-zero exit)
git rev-parse --is-inside-work-tree >/dev/null 2>&1 || { echo "❌ git 저장소 아님 → §4"; exit 2; }
<TEST_CMD>  || { echo "❌ 테스트 실패"; exit 1; }
<LINT_CMD>  || { echo "❌ 린트/타입 에러"; exit 1; }
[ -f "docs/reviews/phase-${N}-review.md" ] \
  && echo "✅ 리뷰 기록 존재" || { echo "❌ phase $N 독립 리뷰 기록 누락 → §3.2"; exit 1; }
```

## 풀 버전과의 차이 요약

| 항목 | 풀 [AGENTS.md](AGENTS.md) | 경량판 |
| --- | --- | --- |
| 아키텍처 산출물 7종 + ADR | 강제 | **면제** (되돌리기 어려운 결정만 3~5줄 메모) |
| 시퀀스/데이터/배포 다이어그램 | mermaid 강제 | **면제** |
| 교차 리뷰어 수 / 감사 추적 | 다중 모델 + 해시·신선도 바인딩 | **독립 1회** + 간단 기록 |
| 독립 교차 리뷰 게이트 | 필수 | **필수 (유지)** |
| 콘솔 증거 강제 | 필수 | **필수 (유지)** |
| 테스트·린트·커버리지 게이트 | 필수 | **필수** (커버리지는 선택) |
| AC 체크박스 자동 강제 | 권장 | **hook/CI 강제 권장 (격상)** |
| 회고 Human Check Items | 정형 표 강제 | **선택, 한 단락** |
